24. Improving Profile Hidden Markov Models for the Detection of Remote Homologs Through Introduction of Phylogenetic Inference

Dawn J. Brooks and Sean R. Eddy[1]

Keywords: hidden Markov models, sequence analysis, phylogenetics, evolutionary models

1     Introduction

Because sequence homology often provides the first clue to protein function, considerable effort has been invested in developing sequence analysis tools that can successfully discriminate between highly divergent homologs and unrelated sequences. Profile hidden Markov models (HMMs), probabilistic representations of the sequences of the members of a protein family, are one such tool. Here we introduce phylogenetic inference and maximum likelihood (ML) models of protein evolution in building HMMs. Others have previously demonstrated that phylogenetic information can be used to improve dramatically the ability to detect remote homologs [3]. Our initial findings confirm these earlier ones. 

2     General Strategy and Significance

In our first approach, we use ML to build target-specific profile HMMs that exploit knowledge of where the target homolog is likely to be within protein family sequence space; rather than building a profile HMM to model the family as a whole, we build one which models the specific target sequence. We expect this application to be particularly useful in cases in which there is strong reason to expect a protein family member to exist, but traditional means for detecting sequence homology have failed to find it. For example, analysis of the complete genome sequences of various organisms suggests that enzymes involved in certain metabolic pathways are either missing or have yet to be correctly annotated. By providing a more sensitive model of the target sequences, we hope to identify existing homologs in many cases where other tools have failed, thus demonstrating the protein to be present. On the other hand, where our model fails to detect a homolog, we will have greater confidence that the homolog truly is absent. As a complementary problem, rather than being unable to detect an expected sequence as above, one might have a sequence (or many sequences, as in the case of genome annotation) for which existing search tools have failed to identify a homolog. In this case, one could search through a database containing a complete set of target-specific profile HMMs for the genome of the query sequence. A significant match between a target-specific profile HMM and the query sequence will indicate the protein family to which the query sequence belongs. Thus, we hope that target-specific profile HMMs will have utility as part of an assembly of computational tools used in annotation of complete genome sequences.

Since the first method relies on general congruence of species and protein family trees, it is not applicable to cases in which the phylogenetic tree representing the protein family is poorly resolved or generally inconsistent with the species phylogenetic tree. Thus, in our second approach, we use ML and simulated evolution to build profile HMMs that attempt to capture sequence features of highly divergent protein family members in general, rather than modeling specific target sequences.

3     Methods

Phylogenetically-based ML sequence inference [4] is used to determine the probability distribution for the identity of the residue in the target sequence (which will be located at an external node on a phylogenetic tree) in the first approach, and the root sequence in the second approach. Since each match state in a profile HMM represents a residue residing within a specific protein environment and subject to a particular evolutionary rate, ML can be used to choose the substitution probability matrix from among several alternatives that best fits (i.e., assigns the greatest likelihood to) the observed data at each match state. Knowledge of the prior distribution of amino acids in the target proteome is used to further refine the model.

In the first method, the probability distribution for each residue, derived as described above, is used directly to estimate the emission probabilities for a match state in the target sequence.  In the second method, this distribution is “evolved” using the matrix determined to best model the evolution of the site in the sequence, in order to represent a more divergent sequence.

Since all amino acids have a non-zero probability of being observed at a given site in the inferred sequence under ML, the method provides a well-justified means to estimate the probability that amino acids not observed at a given position in the alignment might be present in homologous sequences. In this way, we eliminate the need for pseudocounts, substitution matrix or Dirichlet mixtures, which are typically employed in profile HMM parameterization to compensate for small sample size, and frequently to represent features of divergent (undetected) family members at the same time. Another attractive feature of the ML approach is that it inherently weights the contributions of the various descendant sequences in a probabilistic manner, so that more closely related sequences contribute less to the profile probabilities than do more highly divergent ones. This is an improvement over commonly used weighting schemes, which lack formal justification.

4     Preliminary Results

We have found that incorporation of phylogenetic information in building HMMs dramatically improves their performance.  In addition, introducing knowledge of the prior distribution of residues in the target proteome provides significant gains. In initial trials, allowing substitution matrices to vary between sites in the model (according to variations in evolutionary rate and structural environment) resulted in significantly diminished performance, possibly due to overfitting of the data.  We are exploring more sophisticated means by which these refinements in evolutionary modeling may lead to improvements in profile HMMs.

In the future, the performance of our profile HMMs will be compared with that of existing ones, e.g., those used in the current version of HMMER[1] and SAM-T98 [2]. Our profile HMMs, as well as software for creating custom profile HMMs, will be made available online as an extension of the existing Pfam (http://pfam.wustl.edu/) and HMMER (http://hmmer.wustl.edu/) websites.
