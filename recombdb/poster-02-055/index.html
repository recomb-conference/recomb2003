55.  Are there preferences for sequences
longer than triplets in a genome?

D. Whitehead[1] and A. Dress[2]

The large amount of different sequence data has made it clear that organisms have a preference for certain synonomous codons in thier genomes (codon bias). For example this means that GTT is preferred over GTC, which both code for the amino acid valine. There are many possible reasons for this bias, such as mutational bias, high or low GC content, translational preferences, and translational regulation (as some codons are more efficient than others). It is not clear yet how much selection pressure each of these exerts.

But what about sequences longer than triplets? Are there preferences for two codons together, for example. In this work we are looking at all possible combinations of nucleotide sequences (tuples) up to size 9 in sequenced microbial genomes , and looking at the differences between species.

To get an initial feel for the data, a simple distance matrix for each tuple size is created. The distance measure used is a euclidian distance, which is defined as simply the percentage difference for each particular combination, summed together. Although crude, it gives an initial impression.

These distances matrices are visualised by using splitstree. Initial results give basically star trees, except that there is one major partition of the species. This partition is the same for all tuple sizes except oddly enough, three, the codon size.
