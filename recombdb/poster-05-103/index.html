103. RNA Analysis and Antisense Oligodeoxyribonucleotide Prediction Tool


Christian Köberle 1, Volker Patzel 2, Ina Koch 3
Keywords: RNA, oligodeoxyribonucleotide, antisense, JAVA, RMI

Introduction

RNA molecules form complex three-dimensional structures. Complementary parts of the strand of the same RNA molecule are binding to each other. In this way helices are formed. Regions of unpaired nucleotides disrupt helical structures. These unpaired nucleotides are targets for antisense oligodeoxyribonucleotides (ODN) which are binding efficiently to the RNA molecule. ODNs are of great importance for knocking out genes [1]. They can be also used for sensitive detection of special RNAs. One way to find efficiently binding ODNs is shown by Patzel et al. [2] [3] [4]. To find the ODNs they used information about the secondary structure of RNA. They show experimentally a coherence between secondary structure and the kinetics of binding of ODNs to RNA molecules. The result of this investigation was a semi-automatical method to predict ODNs efficiently binding to RNA molecules. The aim of our work was to develop a fully automatic software to predict ODNs - the RNA Analysis amd Antisense Oligodeoxyribonucleotide Prediction Tool (RAOPT).

Development of RAOPT

The prediction of ODNs by Patzel et al. is based on the secondary structure prediction tool Mfold [5], version 2.0 available in HUSAR [6]. Using Mfold on HUSAR the sequence length is limitted to 1400 nucleotides. Thus, to get enough information about the whole structure, the sequence has to be splitted in shorter partitions. These partitions have to overlap in about 1150 nucleotides. The advantage of the partitioning is that one gets more information about the conformation of each nucleotide in the overlapping regions. Furthermore one gets a better assessable computational time for each partition as for the whole sequence.

The fact of partitioning can be used to accelerate the prediction. If a problem could be divided into partitions and each partition could be computed separatly it would be possible to parallelize a program. Our tool RAOPT is implemented in JAVA using the remote method invocation (RMI) technology of JAVA and JAVA-threads. In RAOPT each sequence partition is an own thread, which starts via RMI a secondary structure prediction on a remote computer and returns a fold of this sequence partition to the main-thread. The main-thread combines the information of structure partitions to a consensus prediction of the whole structure. RAOPT is looking in the consensus sequence for longer sequences of unpaired nucleotides. Using rules developed by Patzel et al. [1] [4] RAOPT decides which of these regions of unpaired nucleotides could be involved in the ODN targets using context information.

Results

We have implemented the automatic parallel method for ODN prediction on a LINUX network using Mfold. This tool enables a user to predict ODNs correctly in a reasonable time.

RAOPT can be used as a command line tool or with a graphical user interface (GUI). The user can select sequences from files or input the sequence directly, predict the ODNs, save the results, and view statistical data via the GUI.
