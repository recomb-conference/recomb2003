37. A taxonomical view on the SYSTERS protein family database


Thomas Meinel 1, Antje Krause 2, Martin Vingron 3



Keywords: protein family; taxonomy; multiple alignment

Introduction

The SYSTERS [1] protein family database provides an automatically generated classification of all publicly available protein sequences into disjoint family and superfamily clusters. The underlying sequence set contains all sequences including alternatively spliced isoforms from the SWISS-PROT [5], TrEMBL, and PIR [6] databases as well as protein sequences of several completely sequenced organisms. For each cluster an MView [2] output is generated and from the resulting partial multiple alignment a majority consensus sequence is derived. All consensus sequences together build a searchable sequence database. The sequences in every cluster have been multiply aligned and annotated with known domains from the Pfam database [7].

Taxonomy interface

Every protein sequence in the underlying sequence set of the SYSTERS database belongs to one organism. In terms of taxonomical classification, this is a species, a subspecies or a strain. On the other hand, most protein family clusters contain sequences from several different organisms. Thus, querying the SYSTERS database, one is often interested in
  • the taxonomical complexity of such a protein family,
  • all protein families a specific taxon is represented in,
  • protein families specific for one taxon, or
  • protein families shared by several different taxa.
Here, with taxon we do not only mean a selection on the species level, but on any arbitrary taxonomical level as given by the NCBI taxonomy [4]. To facilitate phylogenetic studies, the SYSTERS web server now provides an interface to select protein family clusters satisfying such a user defined set of taxa.

Interactive multiple alignment

To rely not only on preprocessed multiple alignments of the sequences of one SYSTERS protein family cluster, we extended the functionality of our interface to allow the user to interactively select
  • particular sequences of one SYSTERS cluster (e.g., alternatively spliced isoforms, the consensus sequence, or species specific sequences),
  • sequences from the same SYSTERS superfamily,
  • sequences from other SYSTERS clusters, or
  • own sequences.
This new set of sequences is multiply aligned using the DIALIGN2 program [3]. The resulting alignment is postprocessed by coloring optinally according to the DIALIGN weighting scheme or to the physico-chemical properties of the amino acids similar to the MView program [2].

Access

The SYSTERS protein family database is available for browsing and querying at:
http://systers.molgen.mpg.de
It is possible to enter the hierarchy of protein family clusters and superfamilies at any layer, e.g., by searching for a keyword, choosing a certain taxonomic composition as described above, or by selecting a known Pfam domain. A new protein sequence can be searched against the database of consensus sequences and subsequently multiply aligned together with a user defined selection of sequences from the SYSTERS data set.
